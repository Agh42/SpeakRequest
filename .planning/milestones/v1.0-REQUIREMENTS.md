# Requirements: SpeakRequest — Chair View Redesign (v1.0 Archive)

**Defined:** 2026-03-24
**Completed:** 2026-03-25
**Core Value:** The chair can see every participant's status at a glance — who is speaking, who is next, and who is waiting — without leaving the main screen.

## v1 Requirements

All v1 requirements shipped in milestone v1.0.

### Design System

- [x] **DS-01**: Chair view uses "The Orchestrator" color palette (midnight base `#0b1326`, tonal surface layers, zero explicit 1px borders) — validated in Phase 1
- [x] **DS-02**: Chair view uses Tailwind CSS (CDN) as the style framework, replacing `styles.css` for the chair page — validated in Phase 1
- [x] **DS-03**: Manrope font (headlines/display) and Inter font (body/labels) loaded via Google Fonts in chair.html — validated in Phase 1
- [x] **DS-04**: Material Symbols Outlined icon font used throughout chair view, replacing any previous icon approach — validated in Phase 1

### Layout

- [x] **LAYOUT-01**: Fixed top header bar: hamburger sidebar toggle, room-code badge with copy-code and copy-link buttons, destroy-room button (desktop only), settings icon — completed in Phase 2
- [x] **LAYOUT-02**: Collapsible left sidebar navigation with sections: Meeting Controls, Speaker Queue, Live Polling, Agenda — collapses to 72px icon rail on desktop, slides off-screen on mobile — completed in Phase 2
- [x] **LAYOUT-03**: Desktop layout: 3-column 12-grid — left (4 cols: controls + poll + share), center (5 cols: conference table), right (3 cols: speaker queue) — completed in Phase 2/3
- [x] **LAYOUT-04**: Mobile bottom navigation bar: Controls / Queue / Poll / Menu tabs; sections scroll into view on tap — completed in Phase 2

### Conference Table

- [x] **TABLE-01**: Central round conference table in center column — top-down abstract view, round shape, participants arranged around the perimeter — completed in Phase 3
- [x] **TABLE-02**: Participant avatar displayed as a colored circle with the participant's initials (monogram), color derived from name for visual distinction — completed in Phase 3
- [x] **TABLE-03**: "Speaking now" avatar state: highlighted glowing border in tertiary color (`#00e475`) around the speaking participant's avatar — completed in Phase 3
- [x] **TABLE-04**: "In queue" avatar state: numbered badge (showing queue position 1, 2, 3…) displayed on the avatar — completed in Phase 3
- [x] **TABLE-05**: Center of the table ring shows current speaker timer countdown and room code — completed in Phase 3
- [x] **TABLE-06**: Conference table view is hidden on mobile; mobile layout focuses on controls + queue + poll — completed in Phase 3

### Controls

- [x] **CTRL-01**: Meeting Controls section: speaker-name input field, proxy-request button (add by chair), withdraw button, all wired to existing STOMP events — completed in Phase 4
- [x] **CTRL-02**: Per-speaker time limit input (minutes, numeric), wired to existing `SetLimit` STOMP event — completed in Phase 4
- [x] **CTRL-03**: Timer action buttons: Start, Pause, Next Speaker, Reset — arranged in 2×2 grid, wired to existing `TimerCtrl` STOMP events — completed in Phase 4
- [x] **CTRL-04**: Speaker Queue panel (right column): scrollable list, current speaker card pinned at top with tertiary accent, queued speakers listed below, "Force Add Speaker" button at bottom — completed in Phase 4
- [x] **CTRL-05**: Live Poll glass-panel (left column): displays active poll question, shows live vote-bar results, "End Poll & Record" button — wired to existing poll STOMP events — completed in Phase 4
- [x] **CTRL-06**: Share Access section (left column): copy-code button showing room code, copy-link button — wired to existing sharing logic — completed in Phase 4

### Migration

- [x] **MIG-01**: All existing STOMP subscription handlers (room state, destroyed, error) preserved and wired to updated DOM elements — verified in Phase 5
- [x] **MIG-02**: All existing STOMP publish actions (join as chair, timer controls, speak requests, poll actions, room config) preserved — verified in Phase 5
- [x] **MIG-03**: DOMPurify sanitization applied to all participant names and user-supplied strings rendered into the DOM — verified in Phase 5
- [x] **MIG-04**: Existing room config display (meeting goal, participation format, decision rule, deliverable) retained in settings/sidebar area — verified in Phase 5

## Out of Scope

Explicitly excluded from v1.0.

| Feature | Reason |
|---------|--------|
| Participant view (participant.html) redesign | Deferred — delivered in a later milestone using chair view as reference |
| Popout view (popout.html) changes | Kept as-is; separate concern |
| New backend / API features | UI-only refactor; no new STOMP events or REST endpoints |
| CSS animations and transitions | Keep UI static for now; add in a dedicated animation phase |
| Profile pictures / external avatar images | No profile image support in backend; initials only |
| Agenda management feature | The sidebar Agenda nav item is a placeholder; no backend support yet |
| Auth / login system | Chair identity = WebSocket session; no auth layer added |
