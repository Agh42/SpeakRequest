# Milestone v1.0: Chair View Redesign

**Status:** ✅ SHIPPED 2026-03-25
**Phases:** 1-5
**Total Plans:** 13

## Overview

SpeakRequest v1.0 delivered the chair-view redesign for The Orchestrator: the visual foundation, structural shell, conference table centerpiece, control migration, and final integration verification. The milestone kept the existing STOMP/WebSocket contract intact while replacing the chair UI with a denser, high-end command center.

## Phases

### Phase 1: Design System Foundation

**Goal**: The chair view adopts The Orchestrator visual system so every later UI element can be built on the same tokens, fonts, icons, and surface hierarchy.
**Depends on**: Nothing (first phase)
**Plans**: 4 plans

Plans:

- [x] 01-01: Establish the midnight palette and no-line surface hierarchy
- [x] 01-02: Wire Tailwind CDN, Google Fonts, and Material Symbols baseline
- [x] 01-03: Gap closure for participant/limit/config control styling consistency
- [x] 01-04: Gap closure for remaining poll text input styling consistency

**Details:**
- Established the Orchestrator token set and the tonal midnight surface language.
- Adopted Tailwind CDN and the Google Fonts / Material Symbols baseline in chair.html.
- Closed styling gaps so the chair shell stayed visually coherent for later phases.

### Phase 2: Structural Layout Shell

**Goal**: The redesigned page frame works across desktop and mobile, giving the chair stable navigation and spatial structure before functional panels are migrated.
**Depends on**: Phase 1
**Plans**: 3 plans

Plans:

- [x] 02-01: Build fixed header and desktop three-column shell with anchored section contracts
- [x] 02-02: Implement collapsible sidebar and mobile navigation behaviors with active-section syncing
- [x] 02-03: Close the mobile menu off-screen and active-state scrolling regressions

**Details:**
- Added the fixed header, desktop shell, and anchored section targets.
- Implemented collapsible desktop navigation and mobile bottom tabs.
- Resolved the phase-2 regressions so mobile navigation and active-state highlighting behaved correctly.

### Phase 3: Conference Table Experience

**Goal**: The chair can read participant presence, speaking state, queue order, and active timing from the new central round-table visualization.
**Depends on**: Phase 2
**Plans**: 2 plans

Plans:

- [x] 03-01: Build the desktop conference-table scaffold and mobile hide behavior
- [x] 03-02: Wire participant avatars, speaking highlight, queue badges, and center timing

**Details:**
- Introduced the desktop-only round conference table as the visual centerpiece.
- Rendered initials-only avatars with stable name-derived colors and queue position badges.
- Kept the table hidden on mobile so the shell remained usable on smaller screens.

### Phase 4: Functional Control Migration

**Goal**: Every chair-facing control area is rebuilt in the new layout so the redesign is operational, not just visual.
**Depends on**: Phase 3
**Plans**: 2 plans

Plans:

- [x] 04-01: Rebuild Meeting Controls and Speaker Queue panels while preserving existing chair action IDs
- [x] 04-02: Migrate Live Poll and Share Access, then verify the full control migration in-browser

**Details:**
- Moved the chair control surface into the new shell without changing the existing STOMP destinations.
- Rebuilt the queue, poll, and share panels around the existing DOM/state update model.
- Preserved the room metadata display in the settings/menu area.

### Phase 5: Integration Verification & Completion

**Goal**: The redesigned chair view preserves existing behavior end to end, including subscriptions, publishing actions, sanitization, and room metadata visibility.
**Depends on**: Phase 4
**Plans**: 2 plans

Plans:

- [x] 05-01: Reconfirm chair subscriptions, publish actions, sanitization, and metadata visibility in the migrated shell
- [x] 05-02: Human-verify the migrated chair UI on desktop and mobile before closing the phase

**Details:**
- Reconfirmed the live subscription and publish-action contract inside the migrated chair shell.
- Verified DOMPurify sanitization paths still protect user-supplied strings.
- Human browser approval confirmed the desktop and mobile experience before phase closure.

## Milestone Summary

**Decimal Phases:**

- Phase 2.1: none
- Phase 2.2: none

**Key Decisions:**

- Adopt Tailwind CDN and Google Fonts for the chair redesign to avoid build-pipeline churn.
- Keep the conference table desktop-only so mobile can rely on shell navigation.
- Preserve the existing STOMP destinations, DOM IDs, and room-state snapshot model.
- Keep DOMPurify as the sanitization boundary for all user-supplied content.
- Keep room metadata visible in the settings/sidebar area rather than moving it to a new route.

**Issues Resolved:**

- Rebuilt the chair page into The Orchestrator layout without backend changes.
- Preserved queue, timer, poll, share, and room-metadata behavior while changing the DOM structure.
- Reconfirmed live subscriptions and publish actions after the integration pass.
- Closed the mobile navigation regressions uncovered during shell integration.

**Issues Deferred:**

- Participant view redesign remains deferred to the next milestone.
- Popout view redesign remains deferred.
- CSS animations and transitions remain out of scope for this milestone.
- Profile pictures and external avatar images remain unsupported.
- Agenda management remains a placeholder until a later milestone.

**Technical Debt Incurred:**

- Existing backend tests still use the older `MeetingController(SimpMessagingTemplate)` constructor signature and need follow-up repair.
- Planning artifacts were left behind for earlier gap closures and can be cleaned up in a later housekeeping pass.
