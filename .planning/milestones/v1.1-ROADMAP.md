# Milestone v1.1: UI Improvements

**Status:** ✅ SHIPPED 2026-04-22
**Phases:** 6–11
**Total Plans:** 15

## Overview

SpeakRequest v1.1 extended the shipped chair surface without changing the existing STOMP contract or static-page architecture. The milestone delivered persistent session-keyed room presence, room-title readability fixes, timer urgency colors, a clickable topic label, avatar label truncation, and a full participant-view restyle. All changes build on the v1.0 visual foundation.

## Phases

### Phase 6: Session-Keyed Room Presence

**Goal**: The room state keeps participant presence tied to WebSocket session IDs so the chair surface can show stable avatars until a session disconnects or is replaced.
**Depends on**: Phase 5 (v1.0 foundation)
**Plans**: 2 plans

- Added focused tests: `RoomMemberPresenceTest`, `MeetingControllerPresenceTest`.

## Milestone Summary

**Key Decisions:**

- Session member presence is owned by `Room` (not a parallel roster store) and exposed through snapshots.
- Same-session requests replace the stored member name; chair proxy requests append additional visible entries.
- Timer warning colors apply only to `#speakTimer`; remaining-time display stays at `00:00` after expiry.
- Topic label uses `data-menu-anchor` to scroll to the existing `#section-menu` — no new route.
- Avatar truncation threshold: names ≤5 chars shown in full; longer names → 3-char prefix + `…`.
- Phases 7, 8, 9 were delivered in a single informal commit (aa8e58c) without dedicated GSD phase directories.

**Issues Resolved:**

- Chair proxy requests were collapsing into one queue item — fixed with unique queue IDs per proxy append.
- Avatar label helper had no fallback for empty names — now renders `??` for empty labels.
